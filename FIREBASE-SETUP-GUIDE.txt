## Step 1: Create Firebase Project

### 1.1 Go to Firebase Console
- Open: https://console.firebase.google.com
- Sign in with your Google account

### 1.2 Create New Project
1. Click **"Create a project"** (or **"Add project"**)
2. Enter project name: `portfolio-skills` (or any name you like)
3. Click **"Continue"**

### 1.3 Google Analytics (Optional)
- You can **disable** Google Analytics (not needed for this project)
- Or enable it if you want analytics
- Click **"Continue"** or **"Create project"**

### 1.4 Wait for Setup
- Firebase will create your project (takes ~30 seconds)
- Click **"Continue"** when done

üéâ **Your Firebase project is created!**

## Step 2: Set Up Realtime Database

### 2.1 Navigate to Realtime Database
1. In Firebase Console, look at left sidebar
2. Click **"Build"** section
3. Click **"Realtime Database"**

### 2.2 Create Database
1. Click **"Create Database"** button
2. Choose database location:
   - **United States**: `us-central1`
   - **Europe**: `europe-west1`
   - **Asia**: `asia-southeast1`
   - (Choose closest to you or your visitors)
3. Click **"Next"**

### 2.3 Security Rules
- For now, select **"Start in test mode"**
- This allows read/write for 30 days (we'll secure it later)
- Click **"Enable"**

üéâ **Your database is ready!**

## Step 6: Deploy Your Portfolio

### Option A: GitHub Pages (Free, Recommended)

1. **Create GitHub Repository**
   - Go to https://github.com/new
   - Name it: `portfolio` or `your-username.github.io`
   - Make it Public
   - Click "Create repository"

2. **Upload Files**
   - Click "uploading an existing file"
   - Drag all your files:
     - `skills-section-firebase.html`
     - `admin-login.html`
     - `admin-dashboard-firebase.html`
     - `skills-data-firebase.js`
     - `firebase-config.js`
     - (Plus your other portfolio files)
   - Click "Commit changes"

3. **Enable GitHub Pages**
   - Go to repository Settings
   - Scroll to "Pages" (left sidebar)
   - Source: Deploy from branch
   - Branch: `main` / `root`
   - Click "Save"

4. **Get Your URL**
   - Wait 1-2 minutes
   - Your site will be live at:
   - `https://your-username.github.io/portfolio/skills-section-firebase.html`

### Option B: Netlify (Free, Easier)

1. **Go to Netlify**
   - Open: https://www.netlify.com
   - Click "Sign up" (use GitHub account)

2. **Deploy**
   - Click "Add new site" ‚Üí "Deploy manually"
   - Drag your portfolio folder
   - Wait for deployment (~30 seconds)

3. **Get Your URL**
   - Netlify gives you a random URL like:
   - `https://random-name-12345.netlify.app`
   - You can customize it in Site settings

### Option C: Firebase Hosting (Advanced)

1. Install Firebase CLI: `npm install -g firebase-tools`
2. Login: `firebase login`
3. Init: `firebase init hosting`
4. Deploy: `firebase deploy`

## Step 7: Secure Your Database

**IMPORTANT:** After testing, secure your database!

### 7.1 Update Security Rules

1. Go to Firebase Console
2. Click **"Realtime Database"**
3. Click **"Rules"** tab
4. Replace the rules with:

```json
{
  "rules": {
    "skills": {
      ".read": true,
      ".write": false
    }
  }
}
```

**What this does:**
- ‚úÖ Anyone can READ skills (view portfolio)
- ‚ùå Nobody can WRITE skills (except authenticated users)

### 7.2 Publish Rules
- Click **"Publish"**
- ‚úÖ Your database is now secure!

**Note:** This blocks the admin panel from writing. To fix:
- You can set `.write: true` temporarily when adding skills
- Or implement Firebase Authentication (advanced)

### 7.3 Alternative (Better Security)

```json
{
  "rules": {
    "skills": {
      ".read": true,
      ".write": "auth != null"
    }
  }
}
```

This allows only authenticated users to write.  
(Requires Firebase Authentication setup - not covered in basic guide)

## Troubleshooting

### ‚ùå "Firebase is not defined"

**Solution:**
- Make sure `firebase-config.js` is loaded BEFORE `skills-data-firebase.js`
- Check your HTML file has the correct script order:

```html
<script src="firebase-app-compat.js"></script>
<script src="firebase-database-compat.js"></script>
<script src="firebase-config.js"></script>  ‚Üê This first
<script src="skills-data-firebase.js"></script>  ‚Üê This second
```

### ‚ùå "Permission denied" error

**Solution:**
- Your database rules are too restrictive
- Go to Firebase Console ‚Üí Realtime Database ‚Üí Rules
- Change to test mode temporarily:

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

**Warning:** Don't leave it like this in production!

### ‚ùå Skills not updating in real-time

**Solution:**
- Clear browser cache (`Ctrl+Shift+Delete`)
- Hard refresh (`Ctrl+F5` or `Cmd+Shift+R`)
- Check Firebase Console ‚Üí Database to see if data is there

### ‚ùå "Network request failed"

**Solution:**
- Check internet connection
- Check `databaseURL` in `firebase-config.js` is correct
- Make sure Firebase project is not paused/deleted

## FAQ

### Q: Is Firebase really free?

**A:** Yes! Free tier includes:
- 1 GB database storage
- 10 GB/month download
- 100 simultaneous connections

This is MORE than enough for a portfolio website!

### Q: Will my password be exposed?

**A:** The admin password (`kalle`/`12345`) is in the code, so yes, it's visible.

**Solutions:**
1. Use a random, portfolio-specific password
2. Implement Firebase Authentication (advanced)
3. Accept this limitation for a portfolio demo

### Q: Can I use a custom domain?

**A:** Yes!
- **GitHub Pages:** Settings ‚Üí Pages ‚Üí Custom domain
- **Netlify:** Settings ‚Üí Domain management
- **Firebase Hosting:** `firebase hosting:channel:deploy`

### Q: What if I exceed free tier limits?

**A:** Very unlikely for a portfolio, but:
- Firebase will notify you
- You can upgrade to pay-as-you-go
- Or optimize data usage

### Q: Can I backup my skills data?

**A:** Yes!
1. Go to Firebase Console ‚Üí Realtime Database
2. Click "Export JSON" (top right)
3. Save the file
4. To restore: Import JSON

Or use the "Export Skills" button in the admin dashboard!
